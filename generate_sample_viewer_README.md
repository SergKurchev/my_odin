## 📂 Структура датасета

Все генерации сохраняются в корневую папку `multiview_dataset/`. Каждый сгенерированный куст и его облёты изолированы в собственной подпапке `sample_NNNNN/`.

Структура одной папки сэмпла (`multiview_dataset/sample_00000/`):

```text
sample_00000/
├── cameras.json         # Интринсики и экстринсики всех камер
├── color_map.json       # Расшифровка цветов маски (Instance ID -> Цвет)
├── visualization.html   # (Генерируется скриптом) Интерактивный 3D-просмотр
├── rgb/                 # Фотореалистичные RGB кадры куста
│   ├── 00000.png
│   └── 00019.png
├── depth/               # Точные метрические карты глубины (Z-distance)
│   ├── 00000.npy
│   └── 00019.npy
├── masks/               # Идеальные сегментационные маски без шума
│   ├── 00000.png
│   └── 00019.png
└── labels/              # Bounding Box аннотации (YOLO формат)
    ├── 00000.txt
    └── 00019.txt
```

## 🧬 Форматы данных

### 1. `cameras.json`
Хранит параметры камеры для каждого из 20 (по умолчанию) кадров облёта. 
Содержит **полноценные интринсики** (достаточные для NeRF, TSDF Fusion, COLMAP) и **экстринсики** (позиция в мире Unity).

* **`position`** `[x, y, z]` — Физические координаты камеры в пространстве Unity (Левосторонняя SC, Y-Up).
* **`rotation`** `[x, y, z, w]` — Кватернион поворота камеры в пространстве Unity (local-to-world).
* **`intrinsics`** — Параметры Пинхол-камеры (`fx, fy, cx, cy`, разрешение `width, height`, клипы `near, far` и угол обзора `fov_deg`).

### 2. Карты Глубины (`depth/*.npy`)
Каждая карта сохранена как бинарный `numpy` массив формата `float32`. 
* Значение каждого вложенного пикселя — **Прямое осевое метрическое расстояние (axial depth)** вдоль луча камеры в метрах.
* *Важно:* Сохранены "вверх ногами" из-за специфики видеопамяти Windows (Row 0 = Нижний край изображения), визуализатор автоматически переворачивает их при 3D проекции.

### 3. Маски и `color_map.json`
* **`masks/*.png`** содержит изображения без освещения (Unlit/Flat). Цвет каждого пикселя строго закодирован.
* **`color_map.json`** связывает этот точный цвет со смысловыми данными ягоды:
  * `color`: массив `[R, G, B]`, который нужно искать в `masks/*.png`.
  * `instance_id`: **Глобальный уникальный ID**. Это значит, что ягода №14 в кадре `00000` и ягода №14 в кадре `00019` — **это физически один и тот же меш**.
  * `category_id`: 
    * `0` = Спелая (Ripe)
    * `1` = Неспелая (Unripe)
    * `2` = Полуспелая (Half-ripe)
    * `3` = Цветоножка (Peduncle)

### 4. `labels/*.txt` (YOLO)
Содержит 2D Bounding Boxes всех видимых ягод. 
Формат каждой строки: `<category_id> <x_center> <y_center> <width> <height>` (в относительных координатах `0.0 - 1.0`).

## 🍓 Визуализация (generate_sample_viewer.py)

Скрипт нужен для проверки геометрии, консистентности датасета и инспекции ID ягод. Он собирает все 20 кадров глубины, проецирует их в единое 3D пространство используя `cameras.json`, и строит HTML.

### Использование

```bash
python Scripts/generate_sample_viewer.py multiview_dataset/sample_00000 [опции]
```

### Основные аргументы:
* `sample_path` *(обязательный)* — путь к папке конкретного сэмпла.
* `--mode` — режим фильтрации точек:
  * `plant` **(по умолчанию)** — показывает весь куст (листья, ягоды), вырезая белый фон стен и чёрный пол комнаты.
  * `bush` — оставляет исключительно пиксели ягод (со строгим соответствием маске `masks/*.png`).
  * `all` — рендерит абсолютно все пиксели (куст + стены, полезно для проверки экстринсиков).
* `--stride` *(по умолчанию: 2)* — шаг прореживания пикселей. Укажите `1` для красивых скриншотов, `2` или `4` для скорости.
* `--max-points` *(по умолчанию: 800,000)* — защита браузера от зависания (случайное прореживание).

### Интерфейс в браузере
В левом верхнем углу HTML-файла есть переключатели режимов:
1. **RGB** — Обычный цвет из `rgb/*.png`.
2. **Segmentation** — Раскраска по стадиям созревания (Спелые, Неспелые и т.д.).
3. **Instances** — Окрашивает каждую физическую ягоду в свой уникальный цвет (построен на хешировании `instance_id`). Идеально для проверки того, что сегментация не "мигает" и ID переносятся правильно из кадра в кадр.
